# Архітектурні рішення: повний огляд компонентів

> **Контекст**: Нова Пошта, контакт-центр, 1,000 операторів, 30,000 дзвінків/день
> **Телефонна платформа**: Cisco UCCE 12.6(2) + CUCM 12.6 + CVP 12.6 + Cisco Finesse 12.6
> **Система оператора**: РМ КЦ (Робоче Місце Контакт-Центру) — власна система НП

Цей документ — єдина точка входу для порівняння всіх архітектурних рішень по кожному компоненту системи. Детальні документи по кожному компоненту — у відповідних файлах.

## 1. Копіювання аудіо (Cisco SIPREC)

**Роль**: Cisco під час кожного дзвінка автоматично створює паралельну копію аудіо і відправляє її до AI-систем — оператор і клієнт цього не відчувають. Без цього компоненту жоден AI-сервіс не отримає аудіо. Вибір архітектури тут визначає всі наступні рішення.

**Масштаб**: 30,000 дзвінків/день → ~3,750 одночасних розмов → ~7,500 аудіопотоків → ~240 Mbps.

---

### Варіант А: CCAI Connector (готова інтеграція Cisco + Google)

**Суть**: Cisco UCCE 12.6 має вбудовану підтримку Google AI через готовий посередник. Аудіо автоматично маршрутизується до Google без власного програмного забезпечення. Посередник сам не тарифікується — оплачуються сервіси Google за ним.

**Переваги:**
1. Найшвидший старт — 4–6 тижнів, без розробки сервера
2. Офіційна підтримка Cisco + Google — єдиний варіант з гарантованим SLA від обох вендорів
3. Мінімальна відповідальність команди НП — Google підтримує інфраструктуру

**Ризики:**
1. При одночасному впровадженні Ітерацій 1 і 2 — STT оплачується двічі (~$72K/міс замість $36K), бо Connector передає аудіо лише до Google Agent Assist, а для ACW потрібен окремий потік
2. Підказки оператору відображаються лише у Finesse, не в РМ КЦ
3. Повна залежність від Google — зміна провайдера = переробка архітектури

**Вартість**: одноразово ~$10–20K (налаштування) | STT + Agent Assist ~$54–57K/міс

---

### Варіант Б: Власний сервер (SRS Bridge) ⭐

**Суть**: Мікросервіс на Go/Rust приймає аудіопотік від Cisco і маршрутизує його до будь-якого AI-провайдера: Google STT, OpenAI Realtime, або інших. Один сервер обслуговує одночасно і ACW, і підказки — транскрипт читають обидва компоненти.

**Переваги:**
1. Один потік аудіо на обидві ітерації — STT оплачується один раз (~$36K/міс замість $72K)
2. Підказки відображаються у РМ КЦ — повна гнучкість інтерфейсу
3. Незалежність від одного вендора — можна підключити Google STT, OpenAI або Azure
4. ACW < 15 сек гарантовано — транскрипт готовий одразу в момент завершення дзвінка

**Ризики:**
1. Вища одноразова вартість — $50–90K, 10–16 тижнів розробки
2. Підтримка після запуску — на команді НП

**Вартість**: одноразово ~$50–90K | залежить від вибору STT-провайдера (далі)

> При одночасному впровадженні Ітерацій 1 і 2 Варіант Б економить ~$49K/міс = **~$590K/рік** порівняно з Варіантом А.

## 2. Постобробка і ACW — Ітерація 1 ⭐

**Роль**: Після кожного дзвінка автоматично заповнює картку оператора в РМ КЦ — тема звернення, резюме, disposition-код — за 15 секунд замість 4 хвилин вручну. Вимога: ACW-дані доступні оператору < 15 секунд після завершення дзвінка.

**Залежить від**: Копіювання аудіо (Cisco SIPREC).

---

### Варіант 1: Google STT Chirp 3 + GPT-4o ⭐

**Суть**: Власний сервер (SRS Bridge) слухає аудіо в реальному часі під час дзвінка через gRPC streaming — транскрипт готовий в момент BYE. GPT-4o аналізує текст за 3–5 секунд і відправляє заповнену картку в РМ КЦ. Загальна затримка: 5–8 секунд.
Тип: raw API (Google STT) + raw API (OpenAI) — вся pipeline розробляється командою НП.

**Переваги:**
1. Єдиний хмарний варіант, що надійно відповідає ACW < 15 сек — streaming під час дзвінка, не batch після
2. GPT-4o — найвища якість аналізу тексту: розуміє контекст, тон, нюанси скарг і ескалацій
3. Google STT Chirp 3 — верифікована підтримка uk-UA
4. Спільний STT-потік з Підказками оператору (Ітерація 2) — STT оплачується один раз

**Ризики:**
1. Найвища STT-вартість (~$36,000/міс) — неминуча ціна за streaming; batch-варіанти дешевші, але не вкладаються у 15 сек
2. Вся pipeline розробка — на команді НП (SRS Bridge, промпти, обробка помилок, connector до РМ КЦ)
3. Два контракти: Google + OpenAI

**Вартість**: (SRS Bridge включено до вартості Копіювання аудіо) | щомісяця ~$39,750

---

### Варіант 2: Google STT Chirp 3 + Gemini 2.0 Flash

**Суть**: Та сама архітектура що і Варіант 1, але замість GPT-4o для аналізу тексту — Gemini 2.0 Flash. Повністю Google-стек: один вендор, один контракт.
Тип: vendor-managed (Google) — managed інфраструктура (Cloud Run, Pub/Sub, GCS), нижча one-time розробка.

**Переваги:**
1. Один вендор Google — один контракт, одна команда підтримки
2. Gemini 2.0 Flash значно дешевший за GPT-4o (~$43/міс проти ~$2,250/міс)
3. Нативна GCP-екосистема — менше custom коду, готові тригери і конектори

**Ризики:**
1. Якість аналізу нижча ніж GPT-4o — для простих задач (резюме, тема) достатньо, для QA-аналізу і compliance поступається
2. Повний vendor lock-in Google

**Вартість**: (SRS Bridge включено) | щомісяця ~$37,543

---

### Варіант 3: OpenAI Realtime API (STT + ACW + Підказки в одному потоці)

**Суть**: Власний сервер передає аудіо до OpenAI Realtime API — він одночасно розпізнає мову, генерує підказки під час дзвінка і накопичує транскрипт. Після BYE транскрипт одразу надходить до GPT-4o для ACW. Google STT не потрібен.
Тип: raw API (OpenAI) — вся pipeline розробляється командою НП.

**Переваги:**
1. Один потік аудіо закриває і ACW, і підказки — OpenAI Realtime є і STT, і LLM одночасно
2. GPT бачить повний контекст розмови від початку до кінця
3. Правила НП у system prompt — змінюються без розробки

**Ризики:**
1. Найвища щомісячна вартість — ~$58–87K/міс для STT+LLM разом
2. Вся розробка і підтримка — на команді НП
3. OpenAI не підтримує інтеграцію з Cisco UCCE — питання специфічні до Cisco вирішує команда НП

**Вартість**: (SRS Bridge включено) | щомісяця ~$58–87K

---

### ⛔ Не розглядати: OpenAI Whisper API (batch)

OpenAI Whisper розпізнає текст лише після завершення дзвінка — спочатку зберігається аудіофайл, потім відправляється на сервер. Для дзвінка 2–3 хв: 12–30 сек тільки на розпізнавання + 3–5 сек GPT-4o = **15–35 секунд після BYE**. Вимогу ACW < 15 сек не виконує надійно.
Розглядати лише якщо вимогу змінено до < 40 сек (дешевше: ~$17,250/міс).

## 3. Аналітика і QA — Ітерація 1 ⭐

**Роль**: Автоматичний аналіз 100% дзвінків — скоринг якості, виявлення трендів, coaching операторів. Без Аналітики і QA — контроль якості вибірковий і суб'єктивний.

**Залежить від**: транскрипту з Постобробки і ACW.

---

### Варіант 1: Ender Turing ⭐

**Суть**: SaaS-платформа для QA контакт-центрів. Отримує готовий транскрипт, автоматично оцінює розмову за чеклістом НП, виявляє трендові проблеми, будує coaching-звіти для супервайзерів. Оплата за кількість операторів (per-seat), не за дзвінки.
Тип: vendor-managed SaaS — Ender Turing відповідає за платформу, інфраструктуру і підтримку.

**Переваги:**
1. Фіксована вартість незалежно від кількості дзвінків — при 30K дзвінків/день per-seat значно вигідніший за per-conversation тарифи
2. Готова UI для QA-менеджерів і супервайзерів — не потрібно будувати інтерфейс
3. Вбудований coaching модуль — автоматичні рекомендації операторам
4. Вже інтегрований з популярними contact center платформами

**Ризики:**
1. Якість аналізу uk-UA невідома до POC — потрібен тест на реальних дзвінках НП
2. Умови зберігання даних і SLA — потрібно уточнити до підписання
3. Обмежена гнучкість кастомізації порівняно з власною розробкою

**Вартість**: одноразово ~$10–20K | щомісяця ~$30–60K (до підтвердження на uk-UA POC)

---

### Варіант 2: Google CCAI Insights + Quality AI

**Суть**: Платформа Google для аналізу розмов контакт-центрів. CCAI Insights аналізує теми і тренди, Quality AI автоматично скорить розмови. Нативна інтеграція з BigQuery — всі дані одразу доступні для SQL-запитів і дашбордів.
Тип: vendor-managed (Google) — managed інфраструктура, офіційна підтримка.

**Переваги:**
1. Нативна інтеграція з BigQuery — аналіз даних без ETL, SQL-запити і дашборди одразу
2. Масштабується автоматично — Google Cloud керує інфраструктурою
3. Один вендор якщо решта архітектури на Google

**Ризики:**
1. Тарифікація за кількість аналізованих розмов — при 30K дзвінків/день щомісячна вартість ~$90–160K+ (значно дорожче per-seat Ender Turing)
2. Менш спеціалізована QA-платформа порівняно з Ender Turing — менше готових QA-звітів і coaching-інструментів
3. Складніше налаштувати чекліст НП і бізнес-правила оцінки якості

**Вартість**: одноразово ~$10–20K | щомісяця ~$90–160K+ (при 30K дзвінків/день)

---

### Порівняння варіантів Аналітики

| Критерій | Ender Turing ⭐ | Google CCAI Insights + Quality AI |
|---|---|---|
| Тип тарифікації | Per-seat (фіксована) | Per-conversation (зростає з обсягом) |
| Щомісячна вартість (1K операторів) | ~$30–60K | ~$90–160K+ |
| uk-UA якість | ⚠️ Невідомо — потрібен POC | Верифікована (Google) |

| Coaching модуль | ✅ Вбудований | Базовий |
| Інтеграція з BigQuery | Через export | ✅ Нативна |

## 4. Підказки оператору — Ітерація 2

**Роль**: Поки оператор розмовляє з клієнтом, система за < 2 секунди показує підказку в інтерфейсі: релевантна стаття бази знань, готова відповідь, наступна дія. Мета: AHT -10–15%, FCR +5–10%.

**Залежить від**: Копіювання аудіо. При Варіанті Б (SRS Bridge) — транскрипт вже є з Постобробки і ACW, STT не оплачується повторно.

---

### Варіант 1: Google CCAI Agent Assist

**Суть**: Готовий vendor-managed продукт Google. Кожна репліка клієнта надходить до Agent Assist — він знаходить у базі знань НП релевантну статтю і відправляє підказку оператору у Finesse. Офіційна інтеграція з Cisco UCCE через CCAI Connector.
Тип: vendor-managed — Google відповідає за рішення, інфраструктуру і підтримку.

**Переваги:**
1. Офіційна інтеграція Cisco UCCE + Google — готовий сценарій, задокументований і підтримуваний
2. База знань 5K–50K статей вбудована — Gemini знаходить релевантну статтю і формує відповідь
3. Стабільна затримка < 1 секунди

**Ризики:**
1. При використанні з CCAI Connector — STT оплачується двічі (~$36K додатково), якщо Постобробка на окремому сервері
2. Agent Assist бачить лише останню репліку клієнта — без контексту всієї розмови не розпізнає ескалацію
3. Підказки тільки у Finesse — відображати у РМ КЦ без додаткової інтеграції неможливо
4. Підготовка бази знань займає 4–8 тижнів

**Вартість**: ~$18–21K/міс (на додаток до STT, якщо STT окремий)

---

### Варіант 2: GPT-4o mini (власна розробка) ⭐ при SRS Bridge

**Суть**: Кожна нова репліка надсилається до GPT-4o mini разом з усім діалогом і system prompt з правилами НП. GPT генерує підказку за ~0.5–1 сек. Транскрипт вже є з Постобробки і ACW — STT не оплачується повторно.
Тип: raw API (OpenAI) — вся pipeline розробляється командою НП (WebSocket, RМ КЦ виджет, обробка помилок).

**Переваги:**
1. При SRS Bridge — додаткова вартість лише ~$5–8K/міс (STT вже оплачено Постобробкою)
2. GPT-4o mini бачить повний контекст розмови — розпізнає ескалацію і пропонує компенсацію
3. Правила НП у system prompt — змінюються без участі розробників
4. Підказки у РМ КЦ — через WebSocket у будь-якому інтерфейсі
5. Готовий до пілоту за 1–2 тижні (без підготовки бази знань)

**Ризики:**
1. Вся розробка pipeline — на команді НП (маршрутизація реплік, виджет у РМ КЦ, error handling)
2. Без конкретних статей бази знань — відповіді з system prompt можуть бути менш точними ніж з готової KB
3. Потрібен Enterprise tier OpenAI (~900K запитів/день)

**Вартість**: ~$5–8K/міс (при спільному SRS Bridge з Постобробкою)

---

## 5. Голосовий асистент — Ітерація 3

**Роль**: AI розмовляє з клієнтом до з'єднання з оператором — збирає інформацію, вирішує прості запити, передає контекст оператору. Найдорожчий і найскладніший компонент. Впроваджувати тільки після підтвердження ROI Ітерацій 1 і 2.

**Особливість**: ElevenLabs і інші голосові асистенти **не слухають розмову оператора з клієнтом** — це окрема система, яка розмовляє з клієнтом до з'єднання з оператором.

---

### Варіант 1: ElevenLabs Conversational AI ⭐

**Суть**: Спеціалізована SaaS-платформа для голосових AI-асистентів. Клієнт чує природний голос, веде діалог, AI вирішує прості запити або передає оператору з контекстом.
Тип: vendor-managed SaaS.

**Переваги:**
1. Найбільш природний голос серед усіх варіантів — клієнти не відразу розуміють що говорять з AI
2. Низька one-time вартість впровадження — платформа готова, без власної розробки TTS/ASR
3. Спеціалізований продукт для голосових асистентів — готові інструменти управління діалогом

**Ризики:**
1. Enterprise pricing невідомий — потрібен scoping call для уточнення вартості при 30K дзвінків/день
2. Якість uk-UA мови — потрібен тест на реальних сценаріях НП
3. SLA при такому навантаженні — потрібно уточнити

**Вартість**: одноразово ~$10–20K | щомісяця — до підтвердження Enterprise pricing

---

### Варіант 2: OpenAI Realtime API

**Суть**: API-рівень для голосового AI-діалогу. Власна розробка логіки розмови, TTS/ASR — через OpenAI. Максимальна гнучкість у налаштуванні сценаріїв.
Тип: raw API — вся розробка на команді НП.

**Переваги:**
1. Повна гнучкість сценаріїв — будь-яка логіка діалогу
2. Один OpenAI-контракт якщо решта системи на OpenAI

**Ризики:**
