# Блок 2: AI-асистент до з'єднання з оператором (Pre-Call AI)

> **Пріоритет**: Допоміжний — після підтвердження ROI від Блоків 3 і 4
> **Залежить від**: Блок 1 (копіювання аудіо) — НЕ використовує
> **Пов'язаний з**: РМ КЦ (передача контексту після розмови)

## Що це і навіщо

AI-асистент відповідає на дзвінок клієнта **до того, як до нього підключається живий оператор**. Він ідентифікує клієнта, з'ясовує тему звернення (наприклад, «де моя посилка» або «повернення») і визначає, в яку чергу направити дзвінок. Мета — скоротити час обробки, зменшити навантаження на операторів і збільшити кількість вирішених звернень з першого дзвінка.

> **Чому не перший пріоритет**: Pre-Call AI — найдорожчий компонент із найвищою складністю і найменш передбачуваним ROI на старті. Рекомендується для Фаз 2–3 після того, як Блоки 3 (постобробка/ACW) і 4 (Copilot) підтвердили ROI.

## Рекомендоване рішення ⭐

Рекомендується **Варіант 2 — ElevenLabs**: підтверджено тестуванням НП як найкраща якість розпізнавання та синтезу українського мовлення, швидке впровадження (7–12 тижнів), налаштування без участі розробника. Перед вибором необхідний scoping call з ElevenLabs для підтвердження Enterprise-ціни та SLA при 30K дзвінків на день.

## Варіанти рішень

### Варіант 1: Dialogflow CX (Google)

**Що це:** Готовий продукт Google для IVR — AI-асистент із вбудованою інтеграцією до Cisco CVP (стандартний голосовий шар у системі UCCE НП).

**Переваги:**
1. Нативна інтеграція з Cisco CVP — підключення без додаткового кастомного посередника (моста між системами), економить $20–40K розробки порівняно з Варіантом 3.
2. Детермінований routing: «відстежити посилку» ЗАВЖДИ потрапляє в чергу трекінгу — 100% передбачуваний результат без LLM-сюрпризів.
3. Єдиний вендор якщо НП будує Google-стек для всіх блоків — один контракт, одна команда підтримки.
4. Зрілий enterprise-продукт із документованим SLA, підтвердженим на великих контакт-центрах.

**Ризики:**
1. Найвища фіксована вартість серед варіантів із відомою ціною — ~$81K на місяць при 30K дзвінків.
2. NLU (розуміння мови) потребує навчання: мінімум 200 прикладів на кожен тип запиту, 2–4 місяці до production-якості; специфіка НП (ТТН, КМСКС) — окремо.
3. Зміни в логіці routing можливі лише через розробника — бізнес-команда не може редагувати самостійно.
4. Затримка при зверненні до РМ КЦ для ідентифікації клієнта має бути менше 2 секунд — інакше клієнт чує паузу.

**Вартість:** одноразово ~$20–40K | щомісяця ~$81K

> Vendor-managed продукт — Google відповідає за рішення

### Варіант 2: ElevenLabs ⭐

**Що це:** Спеціалізована AI-платформа для голосових асистентів — підключається до телефонної системи CUCM як звичайний SIP-пристрій (стандартний протокол IP-телефонії) і самостійно веде діалог з клієнтом.

**Переваги:**
1. Найкраща якість uk-UA — підтверджено тестуванням НП на реальних дзвінках. WER (частота помилок розпізнавання) 3.1% на публічному benchmark FLEURS / 5.5% на Common Voice (self-reported, ElevenLabs Scribe v2). Підтримує 4 регіональні акценти: Київ, Львів, Харків, Одеса.
2. Структурована витяжка даних без розробки — до 40 кастомних полів (номер ТТН, тип проблеми, VIP-статус тощо), автоматично через webhook до РМ КЦ. Оператор бачить повний контекст до початку розмови.
3. Підключення через SIP Trunking не змінює існуючу архітектуру UCCE — CUCM маршрутизує дзвінки до ElevenLabs як до будь-якого IP-телефону.
4. Бренд-голос НП через voice cloning — клієнт чує клонований голос бренду НП, а не стандартний синтетичний TTS.
5. Найшвидше впровадження — 7–12 тижнів проти 12–20 для Dialogflow. Немає розробки VXML (скриптів для IVR) і складного навчання NLU.
6. Зміни в логіці IVR через браузер без участі розробника — бізнес-аналітик самостійно редагує питання, поля, правила routing.

**Ризики:**
1. Ціна невідома — Enterprise agreement при 30K дзвінків на день визначається індивідуально. Без scoping call неможливо порівняти з $81K/міс Dialogflow; може виявитись дорожче.
2. Офіційного готового коннектора до UCCE немає — потрібна робота Solutions Engineer ElevenLabs та телефонної команди НП. Не plug-and-play.
3. Молода компанія ($11B оцінка, лютий 2026) — менший досвід на масштабі 30K дзвінків/день порівняно з Google чи NICE.
4. Передача контексту (SIP Transfer) до CUCM 12.6 — стандартний механізм, але потребує тестування з конкретною версією системи НП.

**Вартість:** одноразово ~$15–25K | щомісяця — уточнити (Enterprise pricing)

> Vendor-managed продукт — ElevenLabs відповідає за рішення

### Варіант 3: OpenAI API (власна розробка)

**Що це:** Власноруч побудований міст між телефонною системою CUCM і OpenAI API (GPT-4o) — НП або підрядник розробляє і підтримує всю інтеграційну інфраструктуру самостійно.

**Переваги:**
1. Уніфікований стек якщо НП обирає OpenAI для Блоків 3 і 4 — один вендор, один білінг, розробники знають API.
2. Найгнучкіша бізнес-логіка — всі правила НП описуються текстом у системному налаштуванні GPT-4o. Зміна VIP-routing або нових сценаріїв = редагування тексту без обмежень платформи.

**Ризики:**
1. НП відповідає за всю інфраструктуру — SIP-міст (посередник між телефонією і API), моніторинг, доступність, оновлення — на команді НП або підрядника.
2. Немає вендорної підтримки для інтеграції — OpenAI не допомагає з питаннями «чому SIP transfer не працює з CUCM 12.6».
3. Кастомна розробка SIP-моста — $40–70K одноразово, 4–6 тижнів розробки + постійна підтримка після запуску.
4. Ймовірнісний routing (LLM) — GPT-4o може маршрутизувати несподівано у нетипових сценаріях; для жорстких правил НП це ризик.

**Вартість:** одноразово ~$40–70K (власна розробка!) | щомісяця ~$48–97K (API $43–81K + команда $5–15K)

> ⚠️ Власна розробка — вся відповідальність за підтримку на команді НП

## Рекомендація для пілоту

1. **Не включати в перший пілот** — починати з Блоку 3 (ACW/постобробка): найвищий ROI, найнижча складність.
2. Якщо Pre-Call AI все ж включають — обмежений scope: тільки ідентифікація клієнта та routing, без сценаріїв самообслуговування.
3. Тестова черга: 5–10% вхідних дзвінків через Pre-Call AI з паралельним контролем.
4. Обов'язковий fallback: будь-яка помилка або нерозуміння → негайна передача живому оператору.
5. KPI для оцінки пілоту: клієнт ідентифікований у >80% випадків, routing правильний у >85% випадків.
6. Наступний крок перед вибором ElevenLabs: scoping call із Solutions Engineer ElevenLabs — підтвердження Enterprise pricing, SLA при 30K дзвінків на день і тестування передачі контексту з CUCM 12.6.
