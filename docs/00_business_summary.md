# AI-трансформація контакт-центру Нової Пошти — Зведений аналіз

> **Документ**: Бізнес-аналіз і рекомендації для керівництва
> **Контекст**: 1,000 операторів · 30,000 дзвінків/день · Cisco UCCE 12.6 · власний РМ КЦ
> **Статус**: Технічна специфікація готова. Документ підсумовує рішення, обґрунтування і план дій.

---

## 1. Проблема і ціна бездіяльності

Контакт-центр НП обробляє 30,000 дзвінків на день. При цьому:

| Проблема | Поточний стан | Наслідок |
|---|---|---|
| **ACW після дзвінка** | Оператор вручну заповнює форму 3–5 хвилин | 1,000 операторів × 4 хв = **66 годин/день** непродуктивного часу |
| **QA coverage** | QA-менеджери прослуховують ~1–2% дзвінків вручну | 98% дзвінків не перевіряються. Системні проблеми видно через тижні |
| **Оператор без підказок** | Під час дзвінка оператор шукає інформацію самостійно | Довший AHT, нижчий FCR, клієнт чекає |
| **Аналітика** | Агрегована звітність без деталей по темах і операторах | Рішення приймаються без даних |

**Вартість поточного стану ACW** (консервативна оцінка):
- 30,000 дзвінків/день × 4 хв ACW × $0.25/хв (вартість оператора) = **$650,000/міс**
- При скороченні ACW на 75% → економія **~$487,500/міс**

## 2. Запропонована архітектура — 4 блоки

| Блок | Назва | Пріоритет | Що дає НП |
|---|---|---|---|
| **1** | SIPREC медіа-форкінг | Базовий | Фундамент: дублює аудіо кожного дзвінка для AI-обробки |
| **2** | Pre-Call AI (IVR) | Допоміжний | Голосовий асистент відповідає клієнту до оператора. Знижує навантаження |
| **3** | Post-Call Processing + ACW | **P1 ⭐** | Автозаповнення ACW < 15 сек. Економія ~$487K/міс |
| **4** | Real-time In-Call Copilot | P2 | Підказки оператору під час дзвінка. AHT -10–15% |
| **5** | Analytics + QA | **P1 ⭐** | 100% QA coverage, тренди, coaching для менеджерів |

### Як це працює — крок за кроком

```
 Клієнт телефонує
        │
        ▼
 Pre-Call AI — голосовий асистент [Блок 2, допоміжний]
 "Відстежити посилку? Скарга? Перенаправляю..."
        │
        │  (якщо потрібен оператор)
        ▼
 Cisco CUBE (телефонна станція НП)
        │
        ├──── основний дзвінок ────────────────────────► Оператор (РМ КЦ)
        │                                                      ▲
        │                                                      │ підказки [Блок 4, P2]
        └──── копія аудіо (SIPREC) ──► SRS Bridge             │
                    [Блок 1]                 │                 │
                                     ┌───────┴───────┐        │
                                     ▼               ▼        │
                              Google STT       Google STT ────┘
                              streaming        streaming
                              [Блок 3, P1]     [Блок 4, P2]
                                     │               │
                                     ▼               ▼
                               GPT-4o             Agent Assist
                               аналіз             (підказки)
                                     │
                              ┌──────┴───────┐
                              ▼              ▼
                         ACW у РМ КЦ     BigQuery
                         (< 15 сек)          │
                                             ▼
                                      Analytics + QA
                                      (Ender Turing)
                                       [Блок 5, P1]
```

**Ключова синергія Постобробки і ACW + Підказок оператору**: при спільному впровадженні один сервер обробки (Комбінація B) обслуговує обидва компоненти — розпізнавання мови оплачується один раз (~$36K/міс) замість двох окремих потоків (~$72K/міс). Додатково: GPT-4o mini для підказок (~$5–8K/міс) дешевший за Google Agent Assist (~$18–21K/міс). **Повна економія ~$49K/міс = ~$590K/рік** порівняно з Комбінацією A (CCAI Connector + окремий сервер).

### Блок 3 — Post-Call Processing + ACW ⭐ P1

**Задача**: транскрибувати дзвінок і заповнити ACW у РМ КЦ менш ніж за 15 секунд після завершення розмови.

| | Варіант A ⭐ | Варіант B |
|---|---|---|
| **STT** | Google STT Chirp 3 | OpenAI Whisper API |
| **LLM** | GPT-4o | Gemini Flash |
| **Вартість STT/міс** | ~$36,000 | ~$13,500 |
| **Вартість LLM/міс** | ~$2,250 | ~$43 |
| **Разом/міс** | **~$38,250** | **~$13,543** |
| **ACW latency** | ✅ < 15 сек | ⚠️ ~16–40 сек |

**Чому Варіант A, попри вищу ціну:**
Google STT транскрибує **під час** дзвінка у реальному часі. Коли дзвінок закінчується — транскрипт вже готовий, залишається лише LLM-аналіз (3–5 сек). Разом: 5–8 сек.

OpenAI Whisper працює інакше: спочатку дзвінок закінчується, потім аудіо (2–3 хв запису) відвантажується на сервер і обробляється. Лише це займає 12–30 сек — і ACW-вимога < 15 сек не виконується.

**Щодо LLM**: GPT-4o і Gemini Flash — верифікованих порівнянь на uk-UA немає. Різниця у вартості суттєва ($2,250 vs $43). Рекомендується паралельний тест на 200–500 реальних дзвінках НП до фінального вибору.

## 5. Пілот — план і критерії успіху

### Місяці 1–3: Блок 1 + Блок 3 (P1 core)

**Що робимо:**
1. Розгортаємо SIPREC SRS Bridge (Блок 1) на 10% дзвінків
2. Підключаємо Google STT Chirp 3 streaming
3. LLM (GPT-4o) генерує ACW для тестової групи операторів
4. Валідація: оператори порівнюють auto-fill з тим що написали б вручну

**Критерії успіху:**
- ACW заповнюється < 15 сек після завершення дзвінка
- Точність автозаповнення: операторам не потрібно виправляти > 20% полів
- SIPREC не впливає на якість основного дзвінка (оцінка операторів і клієнтів)
- Uptime pipeline > 99.5%

**Вихід:** 100% дзвінків з auto-fill ACW → економія ~$487K/міс активується

---

### Місяці 4–6: Блок 5 (P1 Analytics + QA)

**Що робимо:**
1. Налаштовуємо BigQuery схему (calls_conversations, calls_analytics, calls_qa)
2. Ender Turing POC: підключаємо 500 архівних дзвінків
3. Порівнюємо Ender Turing QA scores з ручними оцінками QA-менеджерів НП
4. Якщо POC не пройшов (r < 0.75) → оцінюємо Google Insights + Quality AI

**Критерії успіху POC Ender Turing:**
- Кореляція AI score та ручного QA: **r > 0.75**
- 100% дзвінків автоматично оцінені (без пропусків)
- QA-менеджери підтвердили coaching workflow як зручний у роботі
- uk-UA якість скорингу прийнятна (хибні спрацювання < 15%)

**Вихід:** 100% QA coverage замість 1–2% ручного семплування

---

### Місяці 7–9: Блок 4 пілот (P2, 100 операторів)

**Що робимо:**
1. Real-time STT для пілотної групи (100 операторів)
2. Agent Assist + Knowledge Base (~500 статей НП)
3. Copilot widget у sidebar РМ КЦ
4. Логування: які підказки приймаються, які ігноруються

**Критерії успіху:**
- Latency аудіо → підказка на екрані: < 2 сек
- % прийнятих підказок: > 30% (benchmark для нових систем)
- AHT пілотної групи vs контрольної: -5% і більше
- NPS операторів пілотної групи: позитивний

---

### Місяці 10–12: Full rollout

- Розгортання Copilot на всіх 1,000 операторів
- Розширення Knowledge Base до повного обсягу
- Pre-Call AI (Блок 2) — рішення на основі даних P1

## 7. Питання що потребують рішення до старту пілоту

1. **Створити GCP акаунт** — не блокер, але prerequisite. 1 день реєстрації + 3–5 днів на quota request для streaming. Без цього Блоки 1, 3, 4, 5 (BigQuery) не можуть стартувати.

3. **Google чи OpenAI як основний LLM для Блоку 3?**
   Gemini Flash ($43/міс) vs GPT-4o ($2,250/міс) — різниця суттєва. Рекомендується паралельний тест на 500 дзвінках НП до вибору.

4. **Ender Turing чи Google Insights + Quality AI для Блоку 5?**
   Залежить від POC Ender Turing (місяці 4–6). Рішення не потрібне зараз.

5. **Блоки 3 і 4 впроваджувати разом або послідовно?**
   Разом — економія $22,500/міс на STT і менша складність інфраструктури. Але P2 (Copilot) потребує більше часу (20–35 тижнів vs 8–12 для P1). Рекомендація: стартувати P1, паралельно готувати P2 Knowledge Base.

6. **Де розміщувати SIPREC SRS Bridge — on-premise чи GCP?**
   On-premise: нижча latency, дані не виходять за межі НП. GCP: простіше масштабування, менше інфраструктурної відповідальності. Залежить від вимог безпеки НП.

## 8. Висновки і рекомендації

### Загальний висновок

AI-трансформація контакт-центру НП є фінансово обґрунтованою вже на першій фазі. Автоматизація ACW повертає інвестиції за < 1 місяць після launch і дає ~$4M річної економії. Все що після цього (QA, Copilot) — розбудова операційної переваги.

### Рекомендований порядок дій

**Передумова — до будь-яких інших кроків:**
0. **Створити GCP акаунт** — реєстрація займає 1 день. Потрібно: корпоративний email НП, платіжні реквізити, призначити GCP Project Owner. Після реєстрації — одразу запросити збільшення квот на concurrent STT streaming sessions (стандартний процес через Google Cloud Support, займає 3–5 робочих днів).

**Негайно після створення GCP:**
1. Підписати Google Cloud угоду (BAA якщо потрібна відповідність вимогам обробки персональних даних)
2. Сформувати команду розробки: SIPREC Bridge + LLM pipeline (Блок 1 + 3)
3. Почати збір архівних дзвінків для тестування STT і LLM якості

**Паралельно (не блокує старт):**
4. Запросити демо і технічні умови у Ender Turing (включаючи SLA, DPA, BigQuery export)
5. Почати інвентаризацію бази знань НП для Блоку 4 Knowledge Base
6. UX-дослідження з операторами для Copilot widget (Блок 4)

**Не робити зараз:**
- Не підписувати Ender Turing без POC на реальних дзвінках
- Не стартувати Блок 4 до успішного запуску Блоку 3
- Не купувати Google Insights + Quality AI якщо Ender Turing POC пройшов

### Фінальна картина через 12 місяців

| Що отримує НП | Як вимірюється |
|---|---|
| ACW < 15 сек для кожного з 30,000 дзвінків/день | Час від BYE до запису в РМ КЦ |
| 100% QA coverage без збільшення штату QA-менеджерів | % оцінених дзвінків |
| Оператори з підказками в реальному часі | AHT, FCR, % прийнятих підказок |
| Тренди і аномалії видні за 10 хв після дзвінка | Latency analytics |
| Coaching loop без Excel | % операторів що отримали і підтвердили feedback |
