# Блок 5: Post-Call Analytics + QA

> **Пріоритет**: P1 (операційна аналітика) + P3 (глибокий аналітичний модуль)
> **Swimlane**: POST-CALL AI
> **Складність**: Medium
> **Джерело даних**: Блок 3 (транскрипти + LLM-аналіз) → цей блок агрегує, скорить і візуалізує

---

## Роль у архітектурі ToBe

Отримує транскрипти і LLM-аналіз від Блоку 3, виконує глибокий QA-скоринг кожного дзвінка, агрегує тренди і надає дашборди для QA-менеджерів, тренерів і керівництва.

**Два рівні:**
- **P1 — Операційна аналітика**: QA-скоринг, теми, sentiment → доступно через ~10 хв після дзвінка
- **P3 — Глибокий аналітичний модуль**: cohort analysis, предиктивна аналітика, semantic search, BigQuery ML

**Ключова відмінність від Блоку 3**: Блок 3 обробляє один дзвінок і передає в CRM. Блок 5 агрегує всі дзвінки, будує QA-профілі операторів і виявляє системні тренди.

## Архітектура потоку даних

```
Блок 3 (STT Pipeline)
  → Транскрипт JSON + LLM-аналіз (BigQuery + GCS)
        │
        ├─── [P1] QA Scoring Engine ───→ QA Score + Scorecard
        │         (Ender Turing або Custom LLM)
        │
        ├─── [P1] Topic + Sentiment Analytics ──→ Тренди
        │         (CCAI Insights або Custom Pipeline)
        │
        └─── [P3] Custom Analytics Pipeline
                  → BigQuery ML (кластеризація, прогнозування)
                  → Semantic Search (схожі дзвінки)
                  → Looker Studio / Power BI дашборди
```

## Варіанти реалізації

### Варіант 1 ⭐ Рекомендований для пілоту: Custom LLM QA Pipeline + BigQuery

> **Найдешевший старт.** Той самий LLM що обробляє ACW (Блок 3) додатково виконує QA-скоринг. Результати відразу в BigQuery.

#### Переваги в контексті НП

**1. Мінімальна додаткова вартість**
~$500–1,000/міс LLM costs — промпти для QA-скорингу запускаються разом з ACW pipeline у Блоці 3. Фактично безкоштовне розширення існуючого pipeline.

**2. Повна кастомізація під стандарти НП**
QA-чеклист задається у промпті: привітання по стандарту, ідентифікація, вирішення, ескалація, прощання — саме ті пункти які важливі для НП, а не generic шаблон.

**3. Результати одразу в BigQuery**
Єдине сховище для транскриптів, ACW і QA — нативні запити через Looker Studio без ETL.

#### Ризики в контексті НП

| Ризик | Рівень | Деталі |
|---|---|---|
| Немає готового QA UI для менеджерів | Середній | Потрібна розробка дашбордів у Looker Studio або Power BI |
| Калібровка QA-скорів | Середній | LLM-based scoring може давати хибні спрацювання. Потрібна ітерація на 200–500 реальних дзвінках |
| Немає agent coaching UI | Середній | Менеджер бачить цифри, але немає інструменту для прослуховування і коментарів |

**Коли обирати**: пілот P1 — запустити 100% QA coverage якнайшвидше і дешевше. Після підтвердження якості скорів → додати Ender Turing для повноцінного coaching UI.

### Варіант 2: Ender Turing (QA Platform)

> **Рекомендований для scale.** Спеціалізована QA-платформа для контакт-центрів з готовим UI для менеджерів і тренерів.

#### Переваги в контексті НП

**1. Готовий UI для QA-менеджерів без розробки**
Scorecard per call, trending topics, agent leaderboard, coaching flags — все з коробки. QA-менеджер НП відкриває браузер і бачить повну картину.

**2. Per-seat pricing — передбачувані витрати**
$30–60/агент/міс при 1,000 агентах = $30K–60K/міс — фіксована вартість незалежно від кількості дзвінків. На відміну від per-conversation платформ.

**3. 100% автоматичний QA без семплування**
Традиційно QA-менеджери прослуховують 1–2% дзвінків вручну. Ender Turing скорить 100%. Системні проблеми (оператор постійно пропускає ідентифікацію) видно відразу.

**4. Agent coaching workflow**
Менеджер бачить флаги → прослуховує конкретні моменти → залишає коментарі → оператор отримує feedback. Закритий цикл без Excel.

**5. Орієнтований на European/CIS ринки**
Естонська компанія з досвідом українських і European контакт-центрів.

#### Ризики в контексті НП

| Ризик | Рівень | Деталі |
|---|---|---|
| uk-UA якість скорингу — потрібен POC | Середній | Ender Turing використовує власний NLP. Верифікація на реальних дзвінках НП необхідна |
| Висока вартість ($30K–60K/міс) | Середній | Виправдана при ROI від 100% QA coverage і coaching ефективності |
| Залежність від зовнішнього вендора | Низький | SaaS платформа — дані передаються до Ender Turing |

**Коли обирати**: після пілоту з Custom LLM, коли потрібен повноцінний QA workflow і coaching UI для QA-менеджерів НП.

### Варіант 3: Google CCAI Insights

> **Доцільно якщо**: НП вже на Google-first стратегії і BigQuery є центральним сховищем.

#### Переваги в контексті НП

**1. Нативна BigQuery інтеграція**
Результати аналізу прямо у BigQuery → Looker Studio дашборди без ETL.

**2. PhraseMatchers — кастомні теми НП**
Задати теми "Затримка доставки", "Скарга на відділення" як regex/keywords → автоматично тегувати дзвінки.

**3. Generative AI summarization через Gemini**
Якщо в Блоці 3 вже є Gemini — той самий API для analytics summarization.

#### Ризики в контексті НП

| Ризик | Рівень | Деталі |
|---|---|---|
| Найвища вартість | **Критичний** | $72K–126K/міс при 30K дзвінків — значно дорожче Ender Turing ($30K–60K) і Custom LLM (~$1K) |
| Per-conversation ціноутворення | Середній | Вартість росте лінійно з обсягом. При 900K розмов/міс — непрогнозована |

**Коли обирати**: тільки якщо НП стратегічно locked-in на Google і вартість не є пріоритетом.

## P3 — Власний Глибокий Аналітичний Модуль

> Будується після P1 на основі накопичених даних у BigQuery.

```
BigQuery (calls_analytics — 900K дзвінків/міс)
        │
        ├── BigQuery ML:
        │    - k-means clustering: автоматична кластеризація нових тем
        │    - Time series: прогнозування пікового навантаження
        │    - Classification: виявлення складних дзвінків
        │
        ├── Vertex AI Matching Engine:
        │    - Семантичний пошук: "знайди дзвінки схожі на цей"
        │    - Pattern discovery: нові типи скарг що з'являються
        │
        └── Looker Studio / Power BI:
             - QA тренди по операторах, командах, чергах
             - Topic trends (нові проблеми що зростають)
             - CSAT прогнозування через sentiment proxy
             - Cohort analysis: нові vs досвідчені оператори
```

## Складність впровадження

**Рівень: Medium**

| Компонент | Складність | Час |
|-----------|-----------|-----|
| Custom LLM QA pipeline (Фаза 1) | Medium | 4–8 тижнів |
| BigQuery schema + ETL | Medium | 3–6 тижнів |
| Looker Studio / Power BI дашборди | Low | 2–4 тижні |
| Ender Turing інтеграція (Фаза 2) | Low | 2–4 тижні |

- **Adoption QA-менеджерами**: AI-скори мають бути прозорими і поясненими, інакше менеджери не довіряють і не використовують

## Рекомендації для пілоту (MVP scope)

**Тижні 1–4:**
1. Налаштувати BigQuery схему (calls_conversations, calls_analytics)
2. Запустити Custom LLM QA scoring на 500 архівних дзвінках
3. Порівняти AI QA scores з ручними оцінками QA-менеджерів НП
4. Побудувати базовий Looker Studio дашборд

**Критерій успіху:**
- Кореляція між AI QA score та ручним QA: r > 0.75
- 100% coverage (всі дзвінки автоматично оцінені)
- Дашборд доступний QA-менеджерам

**Критерій переходу до Ender Turing (Фаза 2):**
- Custom LLM scoring підтверджено якісним (r > 0.75)
- QA-менеджери потребують coaching workflow, а не лише цифри
