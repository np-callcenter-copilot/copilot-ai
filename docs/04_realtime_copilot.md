# Блок 4: AI-підказки оператору під час дзвінка (Real-time Copilot)

> **Пріоритет**: P2 — після підтвердження ROI від Блоку 3
> **Залежить від**: Блок 1 (копіювання аудіо), Блок 3 (спільне розпізнавання мови)
> **Впливає на**: оператор бачить підказки у РМ КЦ або Finesse

## Що це і навіщо

Поки оператор розмовляє з клієнтом, система в реальному часі (затримка менше 2 секунд) розпізнає мову і показує оператору підказку прямо в його робочому місці — без перемикань, без пошуку. Оператор бачить релевантну статтю бази знань, готову відповідь або наступну дію: наприклад, при скарзі на затримку посилки — одразу пропозицію компенсації.

Ціль: AHT -10–15%, FCR +5–10%.

**Відмінність від Блоку 3**: Блок 3 — обробка запису після завершення дзвінка. Блок 4 — аналіз під час активної розмови, затримка менше 2 секунд.

## Рекомендоване рішення ⭐

Рекомендація залежить від вибору Комбінації у Блоці 1:

- **Комбінація A** (Google CCAI Connector) → **Варіант 1: Google CCAI Agent Assist** — vendor-managed рішення, окремий потік розпізнавання мови (~$36K/міс STT).
- **Комбінація B ⭐** (власний SRS Bridge) → **Варіант 2: GPT-4o mini** — розпізнавання мови вже оплачено Блоком 3, додаткова вартість лише LLM (~$5–8K/міс). Швидший старт, менша загальна вартість.

## Варіанти AI-підказок

### Варіант 1: Google CCAI Agent Assist ⭐ (при Комбінації A)

**Що це:** Готовий vendor-managed продукт від Google — отримує кожну репліку клієнта і повертає підказку з бази знань НП.

**Переваги:**
1. Офіційна інтеграція з Cisco UCCE — backend не потрібно будувати, він готовий.
2. База знань вбудована: 5,000–50,000 статей НП завантажуються у Knowledge Base, Gemini знаходить релевантну статтю і формує готову відповідь оператору.
3. Стабільна затримка менше 1 секунди — аналіз лише останньої репліки, без накопиченого контексту.

**Ризики:**
1. Висока вартість STT: ~$36,000/міс за real-time розпізнавання мови — найвища серед cloud-варіантів для uk-UA.
2. Підготовка бази знань займає 4–8 тижнів: 5,000–50,000 статей uk-UA потрібно структурувати і завантажити до старту.
3. Система бачить лише останню репліку клієнта — якщо клієнт вже третій раз скаржиться на одне й те саме, Agent Assist не розуміє ескалацію.
4. За замовчуванням є ліміти на кількість одночасних сесій — потрібно збільшувати через Google Cloud Support.

**Вартість додатково до STT:** щомісяця ~$18–21K

> Vendor-managed — Google відповідає за рішення, інфраструктуру і підтримку інтеграції з Cisco.

---

### Варіант 2: GPT-4o mini (власні підказки, при Комбінації B ⭐)

**Що це:** LLM-підказки на основі повного тексту розмови — кожна нова репліка надсилається у GPT-4o mini разом з усім діалогом і system prompt з правилами НП.

**Переваги:**
1. Повний контекст діалогу: GPT-4o mini бачить усю розмову від першої репліки. Якщо клієнт третій раз згадує затримку — система розуміє ескалацію і радить компенсацію. Agent Assist побачив би лише останню репліку.
2. Бізнес-логіка НП у system prompt — змінювати без розробки: умови компенсації, VIP-клієнти, процедури для пошкоджених посилок записуються текстом. Зміна правил = редагування тексту, без переналаштування бази знань.
3. Не потрібна база знань: готовий до пілоту за 1–2 тижні замість 4–8.
4. При Комбінації B розпізнавання мови вже оплачено Блоком 3 — додаткова вартість лише GPT-4o mini (~$5–8K/міс).

**Ризики:**
1. Вся pipeline розробка — на команді НП: немає vendor-managed backend. Команда НП будує маршрутизацію реплік, управління промптами, обробку помилок, повторні запити, обмеження навантаження.
2. Якість підказок без реальних статей НП: GPT-4o mini відповідає з system prompt, але без конкретних інструкцій може давати неточні відповіді.
3. Потрібен корпоративний tier OpenAI: ~900,000 запитів/день потребує підвищених лімітів.
4. Зі зростанням контексту протягом тривалого дзвінка затримка може збільшуватись — варто обрізати контекст до останніх N реплік.

**Вартість додатково до STT:** щомісяця ~$5–8K

> ⚠️ Власна розробка pipeline і виджету у РМ КЦ — весь backend і UI команда НП будує самостійно.

## Рекомендація для пілоту

1. Починати з 50–100 операторів (не 1000) — перевірити adoption до масштабування.
2. Тижні 1–4: протестувати якість розпізнавання мови на 100 архівних дзвінках НП — особлива увага до номерів ТТН, назв відділень, імен. Підтвердити затримку менше 1.5 секунди.
3. Тижні 5–8: запустити live розпізнавання мови для 10 операторів без підказок — відображати лише транскрипт. Критерії: затримка менше 1.5 сек, точність розпізнавання краще 12% помилок, uptime 99.5%+.
4. Тижні 9–20: підключити AI-підказки для 100 операторів. Для Agent Assist — пілотна база знань ~500 статей. Для GPT-4o mini — system prompt з основними сценаріями НП.
5. Вимірювати: AHT, FCR, відсоток прийнятих підказок, NPS операторів. ROI пілоту визначає доцільність масштабування на 1000 операторів.
